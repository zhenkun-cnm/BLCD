================================================================================
                    ESHL 电调串口通信协议 — 命令使用手册
================================================================================

【协议概述】

  通信方式  : 半双工 UART, 单线 (USART1, PA9/PA10)
  波特率    : 115200, 8N1
  角色      : 电调始终作为从机, 主机发起所有通信

  包格式 (新—合校验) :
  ┌──────┬──────┬──────┬──────┬────────┬──────────┐
  │ 帧头 │ 地址  │ 长度 │ 命令 │  数据   │ 合校验    │
  │ 1B   │ 2B   │ 1B  │ 1B  │ 0~4B  │ 1B       │
  └──────┴──────┴──────┴──────┴────────┴──────────┘
  最小包长: 5 字节 (无数据段)
  最大包长: 10 字节

  帧头:
    主机→电调: 0xEC
    电调→主机: 0xCE

  地址:
    默认地址  : 0xEC00 (可更改, 存储在内部 Flash 0x0800F800)
    广播地址  : 0xAAAA (改地址命令专用)

【合校验计算方法】

  合校验 = (帧头 + 地址高 + 地址低 + 长度 + 命令 + 数据1 + ... + 数据N) & 0xFF

  即包中除校验字节自身外, 所有字节累加, 取低 8 位。

  示例: 计算 "EC 00 EC 00 06 C3 A1" 的合校验
    EC + 00 + EC + 00 + 06 + C3 + A1 = 0x2E6
    低 8 位 = 0xE6
    完整包: EC 00 EC 00 06 C3 A1 E6

═══════════════════════════════════════════════════════════════════════════════
  0xC1 — 油门命令
═══════════════════════════════════════════════════════════════════════════════

  方向      : 主机 → 电调
  数据格式  : float (4 字节, 大端序)
  数据范围  : 0.0 ~ 1.0 (对应 PWM 80 ~ 991)
  包长      : 10 字节

  HEX 包结构:
    [EC] [AddrH] [AddrL] [0A] [C1] [float_b3] [float_b2] [float_b1] [float_b0] [CHK]

  示例:
    发送 50% 油门 (float 0.5 = 0x3F000000, 大端发送 3F 00 00 00)

    计算合校验: EC + 00 + EC + 0A + C1 + 3F + 00 + 00 + 00 = 0x35C
    取低8位 = 0x5C

    完整包: EC 00 EC 0A C1 3F 00 00 00 5C

  常用油门值:
    0%   → float 0.0 → 00 00 00 00
    25%  → float 0.25→ 3E 80 00 00
    50%  → float 0.5 → 3F 00 00 00
    75%  → float 0.75→ 3F 40 00 00
    100% → float 1.0 → 3F 80 00 00

  说明:
    - 仅在运行状态(RUN_CLOCKWISE / RUN_COUNTER_CLOCKWISE)生效
    - 电机未启动时发送无效
    - 实际 PWM = throttle × 991 (ESHL_RUN_MAX_PWM)


═══════════════════════════════════════════════════════════════════════════════
  0xC2 — 关闭电调
═══════════════════════════════════════════════════════════════════════════════

  方向      : 主机 → 电调 / 电调 → 主机 (回复确认)

  主机→电调:
    数据格式  : uint8_t (填 0x00)
    包长      : 6 字节

    HEX 包结构:
      [EC] [AddrH] [AddrL] [06] [C2] [00] [CHK]

    示例 (地址 0xEC00):
      合校验: EC + 00 + EC + 06 + C2 + 00 = 0x2A6 → 低8位 = 0xA6
      完整包: EC 00 EC 06 C2 00 A6

  电调→主机 (确认关闭):
    HEX: CE 00 EC 06 C2 D0 CHK
    数据 0xD0 表示电调已成功关闭

  说明:
    - 任何状态下均可发送
    - 电调收到后进入 ESHL_STATE_OFF, 关闭所有 MOS 管


═══════════════════════════════════════════════════════════════════════════════
  0xC3 — 启动电调
═══════════════════════════════════════════════════════════════════════════════

  方向      : 主机 → 电调
  数据格式  : uint8_t 方向码
  包长      : 6 字节

  HEX 包结构:
    [EC] [AddrH] [AddrL] [06] [C3] [DIR] [CHK]

  方向码:
    0xA1 = 顺时针 (CW)
    0xA2 = 逆时针 (CCW)

  示例 (地址 0xEC00, 顺时针启动):
    合校验: EC + 00 + EC + 06 + C3 + A1 = 0x348 → 低8位 = 0x48
    完整包: EC 00 EC 06 C3 A1 48

  说明:
    - 仅在 OFF 或 READY 状态生效
    - 启动后依次经历: START → OPEN_LOOP_START → RUN_CLOCKWISE/COUNTER_CLOCKWISE
    - 启动失败会重试最多 5 次
    - 启动 PWM = 80 (ESHL_RUN_MIN_PWM)


═══════════════════════════════════════════════════════════════════════════════
  0xC4 — 电调刹车
═══════════════════════════════════════════════════════════════════════════════

  方向      : 主机 → 电调 / 电调 → 主机 (回复刹车完毕)

  主机→电调:
    数据格式  : uint8_t (填 0x00)
    包长      : 6 字节

    HEX 包结构:
      [EC] [AddrH] [AddrL] [06] [C4] [00] [CHK]

    示例 (地址 0xEC00):
      合校验: EC + 00 + EC + 06 + C4 + 00 = 0x2A8 → 低8位 = 0xA8
      完整包: EC 00 EC 06 C4 00 A8

  电调→主机 (刹车完毕):
    HEX: CE 00 EC 06 C4 A3 CHK
    数据 0xA3 表示刹车已完成

  说明:
    - 仅在运行状态(RUN_CLOCKWISE/RUN_COUNTER_CLOCKWISE)生效
    - 刹车方式由 ESHL_BREAK_MOD 决定: 0=滑行(全关), 1=三相短路刹车
    - 刹车完毕检测: BEMF 过零事件相同次数 ≥ 500


═══════════════════════════════════════════════════════════════════════════════
  0xC5 — 更改电流限制
═══════════════════════════════════════════════════════════════════════════════

  方向      : 主机 → 电调 / 电调 → 主机 (回复确认)

  主机→电调:
    数据格式  : uint16_t ADC 阈值 (大端序, 2 字节)
    包长      : 7 字节

    HEX 包结构:
      [EC] [AddrH] [AddrL] [07] [C5] [LimitH] [LimitL] [CHK]

  电调→主机 (更改成功):
    HEX: CE 00 EC 06 C5 AA CHK
    数据 0xAA 表示更改成功

  ADC 值计算公式:
    ADC = 电流(A) × 0.0005(采样电阻Ω) × 50(放大倍数) × 4096(12位ADC) / 3.3V
        ≈ 电流(A) × 31.03

  示例: 限制 35A → ADC ≈ 1086 → 0x043E
    合校验: EC + 00 + EC + 07 + C5 + 04 + 3E = 0x2F0 → 低8位 = 0xF0
    完整包: EC 00 EC 07 C5 04 3E F0

  说明:
    - 仅在 OFF 或 READY 状态可修改
    - 默认值: 1158 (ESHL_RUN_Current_Limit_ADC_DEFALT)
    - 电流限制值存储在内部 Flash 0x0800F800 附近, 掉电不丢失


═══════════════════════════════════════════════════════════════════════════════
  0xC6 — 电调异常
═══════════════════════════════════════════════════════════════════════════════

  方向      : 电调 → 主机 (电调主动上报)

  数据格式  : uint8_t 异常代码
  包长      : 6 字节

  HEX 包结构:
    [CE] [AddrH] [AddrL] [06] [C6] [ERR_CODE] [CHK]

  异常代码:
    0xE0 — MOS 异常 (MOS 管自检发现短路/开路)
    0xE1 — 电流异常 (运行中电流超过限制)
    0xE2 — 开环启动失败 (重试 5 次无法进入闭环)
    0xE3 — 运行中意外停转 (堵转或负载突变)
    0xE4 — 电池电压异常 (低于额定值 0.1V 以上)

  说明:
    - 电调主动发送, 无需主机请求
    - 同时状态指示灯会闪烁相应的颜色


═══════════════════════════════════════════════════════════════════════════════
  0xC7 — 更改电调地址
═══════════════════════════════════════════════════════════════════════════════

  方向      : 主机 → 电调 (广播) / 电调 → 主机 (广播新地址)

  主机→电调 (广播):
    数据格式  : uint8_t (填 0x00)
    包长      : 6 字节

    HEX 包结构:
      [EC] [AA] [AA] [06] [C7] [00] [CHK]
            ↑─── 广播地址 0xAAAA

    示例:
      合校验: EC + AA + AA + 06 + C7 + 00 = 0x33D → 低8位 = 0x3D
      完整包: EC AA AA 06 C7 00 3D

  电调→主机 (广播自身地址):
    HEX: CE AA AA 08 C7 [NewAddrH] [NewAddrL] CHK
    (电调用广播地址向主机宣告自己的新地址)

  地址设置流程:
    1. 主机广播 0xC7 命令, 所有电调进入 SET_ADDR 状态
    2. 手动快速旋转第一个电调的电机 → BEMF 脉冲触发地址分配 → 地址变为 0xEC00
    3. 状态灯紫色闪烁, 闪烁次数 = 地址 LSB
    4. 电调广播自身新地址, 等待重新上电
    5. 旋转第二个电机 → 地址变为 0xEC01, 以此类推
    6. 全部设置完毕后, 所有电调重新上电

  说明:
    - 必须使用广播地址 0xAAAA 发送
    - 地址设置完毕后需重新上电才能使用新地址
    - 地址存储在 Flash 0x0800F800

  0xC8 — 帮助 (调试命令)
═══════════════════════════════════════════════════════════════════════════════

  方向      : 主机 → 电调
  数据格式  : 无
  包长      : 5 字节 (最小)

  HEX 包结构:
    [EC] [AddrH] [AddrL] [05] [C8] [CHK]

  示例 (广播地址):
    合校验: EC + AA + AA + 05 + C8 = 0x2FD → 低8位 = 0xFD
    完整包: EC AA AA 05 C8 FD

  说明:
    - ★ 调试命令, 跳过合校验 (包尾随便填也不会出错)
    - 电调通过 printf 输出所有命令的简要列表
    - 建议用广播地址 0xAAAA 发送, 任何电调都会响应

  电调 printf 输出示例:
    === ESHL 命令列表 ===
    0xC1 油门(float)    0.0~1.0
    0xC2 关闭(uint8)
    0xC3 启动(uint8)    A1=顺 A2=逆
    0xC4 刹车(uint8)
    0xC5 电流限制(uint16)
    0xC6 异常查询
    0xC7 改地址(广播)
    0xC8 帮助
    0xC9 算合校验(raw)
    =======================


═══════════════════════════════════════════════════════════════════════════════
  0xC9 — 算合校验 (调试命令)
═══════════════════════════════════════════════════════════════════════════════

  方向      : 主机 → 电调
  数据格式  : 原始数据 (你想算校验的那段数据)
  包长      : 可变 (5~10 字节)

  HEX 包结构:
    [EC] [AddrH] [AddrL] [Len] [C9] [raw_data...] [CHK]

  说明:
    - ★ 调试命令, 跳过合校验 (包尾随便填, 不检查)
    - 将你不确定校验值的那段数据放到 payload 里, 电调帮你算合校验
    - 电调通过 printf 返回计算结果

  使用场景:
    你想构建一个"顺时针启动"的包:
      EC 00 EC 06 C3 A1 ??

    不知道 ?? 填什么 → 把目标数据段放入 0xC9 发给电调:

  示例 1 — 计算启动包校验:
    发送: EC 00 EC  09 C9   00 EC 00 06 C3 A1 00
          │         │  │  └──────────────┘ └─ 随便填
          │         │  │  要计算校验的数据段
          │         │  命令 0xC9
          地址+     长度

    电调 printf 返回:
      checksum: [00 EC 06 C3 A1] = 0x48

    于是启动包的校验值就是 0x48:
      EC 00 EC 06 C3 A1 48  ← 完整的启动命令

  示例 2 — 计算油门包校验:
    发送: EC 00 EC 0A C9 00 EC 00 0A C1 3F 00 00 00 00
                                            └─ 随便填

    电调 printf 返回:
      checksum: [00 EC 0A C1 3F 00 00 00] = 0x5C

    于是 50% 油门包:
      EC 00 EC 0A C1 3F 00 00 00 5C

  说明:
    - payload 格式: [目标地址2B] [目标长度1B] [目标命令1B] [目标数据NB]
    - 电调会计算 "目标地址 + 目标长度 + 目标命令 + 目标数据" 的合校验
    - 注意: 不包含帧头 EC 和 payload 中的地址 — 因为目标是算目标包的校验
    - 如果想算包含地址的校验, 把完整数据(含帧头)放进 payload
